# QC.Production_Output_Cost

## Назначение

Физическая таблица с рассчитанной стоимостью производственного выпуска по цехам, номенклатуре и датам. Используется как основной источник данных для аналитики производительности и себестоимости выпуска — показывает сколько выпустили, по какой цене и на какую сумму.

---

## Источники данных

| Источник | Что берём |
|---|---|
| `Views_For_Plan.DailyPlan_CustomWS` | Факт выпуска (`FACT_QTY`) по цехам, датам и номенклатуре |
| `Import_1C.WorkCenter_1C` | Русское название цеха (`WorkShop_Ru`) по `WorkShop_ID` |
| `QC.vw_Stamping_Output` | Список финальных изделий цеха штамповки |
| `Import_1C.vw_Price_List_Current` | Цены на номенклатуру (`PMC_Цеховая` и `PMC_Материальная`) |

---

## Логика расчёта

### Фильтрация цеха штамповки

Цех штамповки (`WorkShopID = 0xB5BC00505601355E11EDF0AED639127E`) обрабатывается особым образом — берутся только **финальные изделия**, то есть те номенклатуры которые присутствуют в `QC.vw_Stamping_Output`. Промежуточные полуфабрикаты (например первая и вторая штамповка одной детали) исключаются чтобы избежать двойного счёта.

Для всех остальных цехов берётся весь выпуск без ограничений.

### Подбор цены

Цена подбирается по номенклатуре и дате выпуска (`OnlyDate`) из `vw_Price_List_Current`:

1. Сначала ищем `PMC_Цеховая`
2. Если не найдена — берём `PMC_Материальная` как fallback

**Логика подбора по дате** — такая же как в других процедурах:
- Берём максимальную цену, дата которой **не позже** даты выпуска (самая свежая актуальная)
- Если таких нет — берём ближайшую **будущую** цену

### Расчёт стоимости

```
Total_Cost = FACT_QTY × Price
```

## Поля таблицы

| Поле | Тип | Описание |
|---|---|---|
| `OnlyDate` | date | Дата выпуска |
| `WorkShopID` | varbinary(16) | ID цеха из 1С |
| `WorkShopName_CH` | nvarchar(255) | Название цеха на китайском |
| `WorkShop_Ru` | nvarchar(255) | Название цеха на русском |
| `NomenclatureID` | varbinary(16) | ID номенклатуры из 1С |
| `NomenclatureNumber` | nvarchar(255) | Артикул номенклатуры |
| `ProductName_CN` | nvarchar(255) | Название продукции на китайском |
| `FACT_QTY` | decimal(18,4) | Фактический выпуск (шт.) |
| `Price` | decimal(18,4) | Цена за штуку |
| `PriceTypeName` | nvarchar(255) | Тип цены (`PMC_Цеховая` или `PMC_Материальная`) |
| `Price_Date` | date | Дата прайса который был использован |
| `Total_Cost` | decimal(18,4) | Итоговая стоимость выпуска (`FACT_QTY × Price`) |
| `Refreshed_At` | datetime2 | Время последнего обновления строки |

---

## Обновление данных

Таблица полностью пересчитывается процедурой `QC.sp_Refresh_Production_Output_Cost` через `TRUNCATE + INSERT`.

Процедура вызывается автоматически в конце каждого цикла импорта `Daily_PlanFact`:

```
1C_PlanFactCopyScript.py  →  EXEC QC.sp_Refresh_Production_Output_Cost
1C_PlanFactOneCopy.py     →  EXEC QC.sp_Refresh_Production_Output_Cost
```
