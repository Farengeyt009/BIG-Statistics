# QC.QC_Repainting_Bom

## Назначение

Физическая таблица для расчёта себестоимости перекраса на производстве. Содержит все строки спецификаций (BOM), связанные с покраской деталей — как сами крашеные детали, так и сборочные изделия, в состав которых эти детали входят. Используется для расчёта затрат на повторную покраску при выявлении брака на различных стадиях производства (штамповка, шелкография и т.д.).

> **Ранее существовала как VIEW.** Переведена в физическую таблицу для повышения производительности — запросы из `QC.QC_Cards_Summary` обращаются к ней многократно для каждой карты брака.

---

## Источник данных

`Import_1C.vw_Import_BOM_Current` — актуальный снимок спецификаций из 1С.

---

## Обновление данных

### Процедура обновления

`QC.sp_Refresh_QC_Repainting_Bom` — выполняет `TRUNCATE TABLE` + `INSERT` с полным пересчётом всех строк и поля `BOM_ID`.

### Когда запускается

Процедура вшита в скрипт импорта BOM из 1С и запускается **автоматически после каждого обновления** таблицы `Import_1C.Import_BOM`.

| Скрипт | Тип | Тайминг |
|---|---|---|
| `1C_BOM/1C_BOM_CopyScript.py` | Автоматический (бесконечный цикл) | Каждые **24 часа** |
| `1C_BOM/1C_BOM_OneCopy.py` | Ручной запуск | По требованию |

### Порядок выполнения в скрипте

```
1. Получить данные из 1С
2. Загрузить в staging (Import_1C.stg_Import_BOM)
3. EXEC Import_1C.sp_SwitchSnapshot_Import_BOM  ← переключение снапшота BOM
4. EXEC QC.sp_Refresh_QC_Repainting_Bom         ← пересчёт этой таблицы
```

## Логика формирования выборки

### Шаг 1 — Крашеные детали (Часть 1)

Отбираем все BOM, у которых:
- `GPNomencl_No LIKE '8.75.11%'` — готовая продукция является крашеной деталью (артикул начинается с 8.75.11)
- `Nomencl_MidleGroupID = 0xB3C5C4CBE1AC069511EE9898D9D5ABD1` — строка материала относится именно к краске (фильтр по средней группе номенклатуры)

Таким образом получаем строки спецификаций крашеных деталей, где в качестве материала указана непосредственно краска. Это позволяет точно знать расход краски на каждую деталь.

### Шаг 2 — Сборочные изделия с крашеными деталями (Часть 2)

Добавляем строки BOM, у которых:
- `Nomencl_No LIKE '8.75.11%'` — материалом в спецификации является крашеная деталь (то есть в изделие входит деталь с артикулом 8.75.11)

Это необходимо для расчёта себестоимости брака, выявленного уже после покраски — на стадии шелкографии, сборки и других последующих операциях. В таком случае нужно знать не только стоимость самой краски, но и стоимость перекраса детали как составляющей готового изделия.

Итоговый набор строк — объединение обеих частей через `UNION ALL`.

## Поле BOM_ID

Ключевое вычисляемое поле, которое связывает каждую строку выборки с конкретной спецификацией крашеной детали (`Spec_ID`). Используется для группировки и расчётов в `QC.QC_Cards_Summary`.

### Логика подстановки:

**Случай 1 — строка сама является крашеной деталью** (`GPNomencl_No LIKE '8.75.11%'`):

`BOM_ID = Spec_ID` текущей строки. Никакого поиска не нужно — строка уже принадлежит BOM крашеной детали.

**Случай 2 — строка является сборочным изделием** (`Nomencl_No LIKE '8.75.11%'`):

Ищем `Spec_ID` крашеного BOM, который соответствует данному изделию. Поиск ведётся только внутри `BOM_Filtered` (то есть только среди крашеных деталей с правильным `Nomencl_MidleGroupID`), что исключает попадание посторонних BOM.

Условия поиска:
1. `GPNomencl_ID = Nomencl_ID` текущей строки — связь: крашеная деталь входит в наше изделие
2. `Delete_Mark = 0x00` — BOM не помечен на удаление
3. Временной диапазон BOM крашеной детали пересекается с диапазоном текущего BOM (дата начала крашеного BOM не позже конца текущего, и не позже сегодня)
4. Среди подходящих — берём с максимальной `Start_Day` (самый актуальный)

**Fallback (если по условиям диапазона ничего не найдено):**

Берём любой BOM крашеной детали для данного изделия с максимальной `Start_Day`, игнорируя временные диапазоны. Это страховка для случаев, когда даты BOM не совпадают идеально, но связь между деталями однозначно существует.

## Выводимые поля

| Поле | Описание |
|---|---|
| `Spec_ID` | ID спецификации (BOM) |
| `GPNomencl_ID` | ID готовой продукции |
| `Nomencl_ID` | ID номенклатуры материала |
| `Labor_TypeID` | ID вида трудозатрат |
| `Delete_Mark` | Пометка удаления (0x00 = активен) |
| `Spec_Status` | Статус спецификации (порядковый номер перечисления) |
| `Start_Day` | Дата начала действия BOM |
| `Finish_Day` | Дата окончания действия BOM (`0001-01-01` = бессрочно) |
| `BOM_No` | Код спецификации |
| `GPNomencl_No` | Артикул готовой продукции |
| `GPMaterial_Name` | Наименование готовой продукции |
| `Nomencl_No` | Артикул материала |
| `Material_Name` | Наименование материала |
| `GPMaterial_Labor` | Норма трудозатрат на готовую продукцию |
| `Porebnost` | Потребность (норма расхода материала) |
| `BOM_ID` | ID спецификации крашеной детали (вычисляемое при обновлении, см. логику выше) |
