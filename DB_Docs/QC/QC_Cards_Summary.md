# QC.QC_Cards_Summary

## Назначение

Сводная таблица карт брака с рассчитанной стоимостью по каждому дефекту. Используется как основной источник данных для аналитики по браку на производстве — показывает кто, когда, что забраковал и сколько это стоит.

---

## Источники данных

| Источник | Что берём |
|---|---|
| `Import_1C.vw_QC_Cards_Current` | Основа — все карты брака из 1С |
| `Import_1C.vw_Order_1C_v2_Current` | Рынок, артикул и название готовой продукции по производственному заказу |
| `Ref.Product_Guide` | Большая группа и группа товара по артикулу |
| `QC.QC_Repainting_Bom` | Спецификации для расчёта стоимости перекраса (`QCcardConclusion_No = 0`) |
| `Import_1C.vw_Price_List_Current` | Цены на материалы и готовую продукцию |
| `Import_1C.vw_Labor_Cost_Current` | Ставки стоимости работ по видам трудозатрат |

---

## Логика расчёта стоимости

Каждая карта брака имеет заключение (`QCcardConclusion_No`). В зависимости от него стоимость считается по-разному.

---

### Заключение = 0 — Перекрас

Самый сложный случай. Деталь нужно перекрасить заново, поэтому считаем реальную стоимость материалов (краска) и работ (покраска).

**Материальная себестоимость (`Material_Cost`):**

1. Берём `QC_Card_NomenclatureID` (ID забракованной детали) и находим её спецификацию в `QC_Repainting_Bom` — ту, чей период действия совпадает с датой создания карты брака
2. По найденной спецификации определяем какие краски и в каком количестве (`Porebnost`) используются
3. Для каждой краски находим цену в прайсе (`PriceTypeName = 'БюджетнаяЦена'`) на дату создания карты брака
4. Итог: `SUM(расход краски × цена краски)` по всем краскам

**Стоимость работ (`Labor_Hours`):**

По той же спецификации берём трудоёмкость и ставку по виду трудозатрат (`vw_Labor_Cost_Current`) на дату карты. Итог: `SUM(трудоёмкость × ставка)` по всем операциям.

**Итоговая стоимость (`Labor_Cost`):**

`Material_Cost + Labor_Hours`

---

### Заключения = 1, 2, 3 — Другие виды брака

Более простой расчёт — берём готовые цены из прайса по артикулу забракованной детали.

**Материальная себестоимость (`Material_Cost`):**
Цена из прайса с типом `PMC_Материальная` на дату карты брака.

**Стоимость работ (`Labor_Hours`):**
Цена `PMC_Цеховая` минус цена `PMC_Материальная` на дату карты брака.

**Итоговая стоимость (`Labor_Cost`):**
Цена из прайса с типом `PMC_Цеховая` на дату карты брака.

---

### Логика подбора цены по дате

Для всех типов заключений используется одинаковый принцип подбора цены:
- Берём максимальную цену, дата которой **не позже** даты создания карты брака (самая свежая актуальная цена)
- Если таких цен нет — берём ближайшую **будущую** цену (страховка для ранних карт)

Таблица содержит карты брака начиная с **2025-01-01**.

## Обновление данных

| Скрипт | Тип | Тайминг |
|---|---|---|
| `1C_QCCards/1C_QC_Cards_CopyScript.py` | Автоматический | Каждые **10 минут** |
| `1C_QCCards/1C_QC_Cards_OneCopy.py` | Ручной запуск | По требованию |

### Порядок выполнения в скрипте

```
1. Получить карты брака из 1С (последние 60 дней)
2. Загрузить в staging (Import_1C.stg_QC_Cards)
3. EXEC Import_1C.sp_SwitchSnapshot_QC_Cards  ← переключение снапшота карт брака
4. EXEC QC.sp_Refresh_QC_Cards_Summary        ← пересчёт этой таблицы
```
